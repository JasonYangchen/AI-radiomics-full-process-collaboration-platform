# AI+影像组学全流程协作平台 - 产品需求文档

## 产品名称
RadiomicsHub - AI+影像组学全流程协作平台

## 产品概述
面向医学影像团队的协作平台，支持多用户登录、影像上传、感兴趣区域标注、特征提取、机器学习建模全流程。平台采用角色权限分离机制，管理员负责数据管理和特征提取，医生专注于影像标注工作。

## 核心功能

### 1. 用户认证与权限管理
- 多用户注册/登录系统
- 角色区分：管理员(Admin)、医生(Doctor)
- JWT Token 认证
- 用户管理（管理员专属）

### 2. 影像上传与管理
- 支持格式：DICOM (.dcm)、NRRD (.nrrd)、NIfTI (.nii/.nii.gz)
- 批量上传功能
- 影像元数据解析
- 影像预览（2D切片、3D渲染）
- 影像归档与分类

### 3. 影像标注工具
- 类似 3D Slicer 的标注体验
- 支持多种标注工具：
  - 自由画笔
  - 多边形套索
  - 球体/立方体
  - 阈值分割
- ROI 自动保存
- 标注历史版本
- 多医生协作标注
- 标注一致性校验

### 4. 特征提取（管理员专属）
- 集成 PyRadiomics 库
- 支持特征类别：
  - 一阶统计特征
  - 形状特征
  - 纹理特征 (GLCM, GLRLM, GLSZM, NGTDM)
  - 高阶特征
- 特征参数配置
- 批量特征提取
- 特征结果导出（CSV/Excel）

### 5. 机器学习建模
- 数据集划分（训练/验证/测试）
- 特征标准化/归一化
- 模型选择：
  - 逻辑回归
  - 随机森林
  - 支持向量机
  - XGBoost
  - 深度学习模型
- 超参数调优
- 交叉验证
- 模型性能评估：
  - 准确率、敏感度、特异度
  - ROC曲线、AUC值
  - 混淆矩阵
  - 校准曲线

### 6. 可视化报告
- 模型性能可视化图表
- 特征重要性排序
- 预测结果展示
- 报告导出（PDF）

## 用户故事

### 管理员用户
1. 作为管理员，我希望能批量上传医学影像，以便快速建立影像数据库
2. 作为管理员，我希望能提取影像特征，以便后续建模分析
3. 作为管理员，我希望能管理用户账号，以便控制平台访问权限
4. 作为管理员，我希望能创建和训练模型，以便进行预测分析

### 医生用户
1. 作为医生，我希望能浏览影像列表，以便选择需要标注的影像
2. 作为医生，我希望能使用多种工具标注ROI，以便准确勾画感兴趣区域
3. 作为医生，我希望能保存和修改标注，以便完善标注结果
4. 作为医生，我希望能查看自己的标注统计，以便了解工作进度

## 验收标准

### 功能验收
- [ ] 用户能正常注册、登录、退出
- [ ] 管理员能上传 DICOM/NRRD 格式影像
- [ ] 医生能在 3D 视图中进行 ROI 标注
- [ ] 标注结果能自动保存到数据库
- [ ] 管理员能成功提取影像特征
- [ ] 特征提取结果可导出为 CSV
- [ ] 能完成模型训练并生成评估报告
- [ ] 可视化图表正确显示

### 性能验收
- 单个影像上传 < 30秒
- 标注操作响应 < 500ms
- 特征提取 < 5分钟/影像
- 模型训练 < 30分钟（100例数据）

### 安全验收
- 密码加密存储
- API 接口鉴权
- 敏感操作日志记录
- 数据传输 HTTPS 加密

## 技术约束

### 必须满足
1. 使用 Docker 容器化部署
2. 使用 PyRadiomics 进行特征提取
3. 支持 DICOM/NRRD 格式
4. 前后端分离架构

### 技术选型
- 前端：React + TypeScript + VTK.js/ITK.js（3D渲染）
- 后端：FastAPI（Python）
- 数据库：PostgreSQL
- 存储：MinIO（S3兼容对象存储）
- 缓存/队列：Redis + Celery
- 部署：Docker Compose
